## Purpose

This document defines the functional user stories for Auan-Auan-Platform.

Each story describes the expected behavior from the perspective of an end user.

---

## Story Format

Every user story follows:

```text
As a <role>

I want <goal>

So that <benefit>
```

---

## Customer Stories

### Customer Registration

```text
As a customer

I want to sign in using LINE Login

So that I can start ordering without creating another account.
```

---

### Browse Categories

```text
As a customer

I want to browse food categories

So that I can quickly find the products I want.
```

---

### Search Products

```text
As a customer

I want to search for products

So that I can find menu items faster.
```

---

### View Product Details

```text
As a customer

I want to view detailed product information

So that I can make informed purchasing decisions.
```

---

### Select Product Options

```text
As a customer

I want to select required and optional product options

So that my order matches my preferences.
```

---

### Add Product To Cart

```text
As a customer

I want to add products to my shopping cart

So that I can purchase multiple items together.
```

---

### Update Cart

```text
As a customer

I want to update item quantities

So that I can adjust my order before checkout.
```

---

### Remove Cart Item

```text
As a customer

I want to remove products from my cart

So that I can change my order.
```

---

### Checkout

```text
As a customer

I want to review my order before payment

So that I can verify everything is correct.
```

---

### Enter Delivery Address

```text
As a customer

I want to provide my building and room number

So that my order can be delivered correctly.
```

---

### PromptPay Payment

```text
As a customer

I want to pay using PromptPay QR

So that I can complete my payment quickly.
```

---

### Payment Confirmation

```text
As a customer

I want to notify the store after payment

So that my order can be processed.
```

---

### Track Order

```text
As a customer

I want to view my order status

So that I know when my food is ready.
```

---

### Order History

```text
As a customer

I want to view my previous orders

So that I can review my purchases.
```

## Owner Stories

### View Dashboard

```text
As the owner

I want to see today's business summary

So that I can monitor operations.
```

---

### Manage Products

```text
As the owner

I want to create, update, and disable products

So that the menu remains accurate.
```

---

### Manage Categories

```text
As the owner

I want to organize products into categories

So that customers can browse easily.
```

---

### Receive New Orders

```text
As the owner

I want to receive notifications for new orders

So that I can begin preparing food immediately.
```

---

### Verify Payments

```text
As the owner

I want to verify customer payments

So that only paid orders enter preparation.
```

---

### Update Order Status

```text
As the owner

I want to update order status

So that customers know the progress of their orders.
```

---

### View Customers

```text
As the owner

I want to view customer information

So that I can better support repeat customers.
```

---

### View Sales

```text
As the owner

I want to review sales reports

So that I understand business performance.
```

- Product Management

## Definition of Done

User stories are complete when:

- Every major user role is covered.
- Every MVP feature has a corresponding story.
- Stories describe user value.
- Stories support future expansion.
