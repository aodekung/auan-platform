# Phase 3: Upload Features — Implementation Plan

## สิ่งที่ค้นพบ (ดีมาก)

Backend สำหรับ **Logo** และ **QR Code** upload มีอยู่แล้ว 80%:
- `@fastify/multipart` ✅ installed + registered
- `POST /admin/settings/store/logo` ✅ route พร้อม (save → `uploads/store/`)
- `POST /admin/settings/payment/qrcode` ✅ route พร้อม (save → `uploads/payment/`)
- `saveUploadedFile()` helper ✅ (MIME validate, size check, UUID filename)
- `store.logo` + `payment.promptpay_qr` ✅ setting keys มีใน constants

**ที่ขาด:**
1. `@fastify/static` ยังไม่ติดตั้ง → ไฟล์ที่ upload ไม่สามารถเข้าถึงผ่าน HTTP ได้
2. Admin UI ไม่มี upload component ใน Settings page
3. Product image upload route ยังไม่มี
4. Sidebar ใช้ hardcoded "Auan-Auan" + Flame icon

---

## Step 3.0: Static File Serving (prerequisite สำหรับทุกอย่าง)

**ไฟล์:** `apps/api/package.json`, `apps/api/src/server.ts`

1. `pnpm add @fastify/static --filter @auan/api`
2. Register static serving ใน `server.ts`:
   ```ts
   import fastifyStatic from "@fastify/static"
   import path from "path"
   // After multipart registration:
   await app.register(fastifyStatic, {
     root: path.join(process.cwd(), env.UPLOAD_PATH),
     prefix: "/uploads/",
     maxAge: 86400000, // 1 day cache
   })
   ```
3. Result: ไฟล์ที่ `uploads/store/xxx.png` สามารถเข้าถึงได้ที่ `/uploads/store/xxx.png`

## Step 3.1: Logo Upload UI + Display

### 3.1a — Admin Settings > Store Info: เพิ่ม Upload UI

**ไฟล์:** `apps/admin/src/hooks/use-settings.ts`
- เพิ่ม `useUploadStoreLogo()` mutation hook
  - `POST /admin/settings/store/logo` ด้วย `FormData` (multipart)
  - `onSuccess`: invalidate settings queries

**ไฟล์:** `apps/admin/src/pages/settings-page.tsx` — `StoreInfoCard`
- เพิ่ม image upload section ด้านบน form (ก่อนชื่อร้าน):
  - ถ้ามี logo → แสดง preview + ปุ่ม "เปลี่ยนรูป"
  - ถ้ายังไม่มี → แสดง dashed border upload zone + "อัพโหลดโลโก้"
  - ใช้ `<input type="file" accept="image/png,image/jpeg,image/webp">` hidden, trigger ผ่าน button

### 3.1b — Admin Sidebar: Dynamic branding

**ไฟล์:** `apps/admin/src/components/admin/sidebar.tsx`
- เรียก `useSettingsByCategory("store")` hook
- อ่าน `store.name` และ `store.logo` จาก settings map
- ถ้ามี logo URL → render `<img>` แทน `Flame` icon
- ถ้าไม่มี → fallback ใช้ `Flame` icon (เหมือนเดิม)
- Store name ใช้จาก settings แทน hardcoded "Auan-Auan"

### 3.1c — Admin Login page: แสดง logo (optional)

**ไฟล์:** `apps/admin/src/pages/login-page.tsx`
- เรียก `useStoreSettings()` hook
- ถ้ามี logo → แสดง `<img>` แทน `Flame` icon
- ถ้าไม่มี → fallback Flame icon (เหมือนเดิม)

## Step 3.3: Product Image Upload

### 3.3a — Backend: Product image upload route

**ไฟล์:** `apps/api/src/modules/products/products.service.ts`
- เพิ่ม `uploadProductImage(id: string, file: MultipartFile)` function
- Extract `saveUploadedFile` logic → create `apps/api/src/common/upload.ts` (shared utility)
- Save to `uploads/products/`, update `product.imageUrl` in DB

**ไฟล์:** `apps/api/src/modules/products/products.controller.ts`
- เพิ่ม `uploadProductImageHandler`

**ไฟล์:** `apps/api/src/modules/products/products.schema.ts`
- เพิ่ม route schema for `POST /admin/products/:id/image`

**ไฟล์:** `apps/api/src/modules/admin/admin.routes.ts`
- Register `POST /api/v1/admin/products/:id/image` route

### 3.3b — Frontend: Upload in Product Form

**ไฟล์:** `apps/admin/src/hooks/use-products.ts`
- เพิ่ม `useUploadProductImage()` mutation hook
  - `POST /admin/products/{id}/image` ด้วย `FormData`

**ไฟล์:** `apps/admin/src/pages/products-page.tsx` — `ProductFormDialog`
- เพิ่ม image upload area ด้านบน form:
  - Create mode: upload zone
  - Edit mode: preview existing image + upload/change button
  - ใช้ 2-step flow: create product first → upload image (เพราะต้องมี product ID ก่อน)

### 3.3c — Customer: ไม่ต้องแก้
- `ProductCard` แสดง `imageUrl` อยู่แล้ว (ถ้ามี → img, ถ้าไม่มี → placeholder)
