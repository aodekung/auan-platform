### กลุ่ม B: Backend Fixes (2 อย่าง)

#### B1: Register `@fastify/multipart` plugin
- **สร้าง `apps/api/src/plugins/multipart.ts`**:
  ```typescript
  import fastifyMultipart from '@fastify/multipart'
  import { env } from '../config/env.js'
  export async function registerMultipart(app) {
    await app.register(fastifyMultipart, { limits: { fileSize: env.UPLOAD_MAX_SIZE } })
  }
  ```
- **`apps/api/src/server.ts`**: เพิ่ม `await registerMultipart(app)` ก่อน route registrations
- **ผล:** logo upload + QR upload ทำงานได้

#### B3: Wire LINE push notifications
- **`apps/api/src/modules/notifications/notification.service.ts`**:
  - ใน `processQueue()`: ก่อน mark SENT → เช็ค channel === "LINE" → เรียก `pushMessage()` จาก `lib/line-messaging.ts`
  - try/catch: ถ้าส่ง LINE ล้มเหลว → mark FAILED + log error (ไม่ crash)
- **`apps/api/src/modules/orders/orders.service.ts`**:
  - ใน `updateOrderStatus()`: หลัง transaction → `dispatchNotification(customer.lineUserId, "ORDER_STATUS_UPDATED", "LINE", {...})` สำหรับสถานะ PREPARING, READY, OUT_FOR_DELIVERY, DELIVERED
  - ใน `cancelOrder()`: หลัง transaction → `dispatchNotification(customer.lineUserId, "ORDER_CANCELLED", "LINE", {...})`
- **`apps/api/src/modules/payments/payments.service.ts`**:
  - ใน `verifyPayment()`: หลัง transaction → `dispatchNotification("PAYMENT_VERIFIED", "LINE", {...})`
  - ใน `rejectPayment()`: หลัง transaction → `dispatchNotification("PAYMENT_REJECTED", "LINE", {...})`
- **ทุก notification call ใช้ try/catch** — ถ้า LINE API ล้มเหลว ไม่กระทบ business logic

### กลุ่ม C: Infrastructure Hardening (6 อย่าง)

#### C1: PostgreSQL tuning
- **`docker-compose.prod.yml`**: เพิ่ม `command:` ใน postgres service:
  ```yaml
  command: >
    postgres
    -c shared_buffers=256MB
    -c work_mem=4MB
    -c effective_cache_size=512MB
    -c max_connections=100
    -c random_page_cost=1.1
  ```

#### C2: JWT_COOKIE_SECURE production enforcement
- **`apps/api/src/config/env.ts`**: เพิ่ม `.refine()` บน `JWT_COOKIE_SECURE`:
  ```typescript
  JWT_COOKIE_SECURE: z.coerce.boolean().default(false).refine(val => {
    if (process.env.NODE_ENV === 'production' && !val) return false
    return true
  }, { message: "JWT_COOKIE_SECURE must be true in production" })
  ```

#### C3: Docker resource limits
- **`docker-compose.prod.yml`**: เพิ่ม `deploy.resources.limits` ใน api + nginx:
  ```yaml
  api:
    deploy:
      resources:
        limits: { memory: 512M, cpus: '1.0' }
  nginx:
    deploy:
      resources:
        limits: { memory: 128M, cpus: '0.5' }
  postgres:
    deploy:
      resources:
        limits: { memory: 1G, cpus: '1.0' }
  ```

#### C4: Docker log rotation
- **`docker-compose.prod.yml`**: เพิ่ม `logging:` ในทุก service:
  ```yaml
  logging:
    driver: json-file
    options:
      max-size: "10m"
      max-file: "3"
  ```

#### C5: Nginx CSP header + error pages
- **`nginx/nginx.conf`**: เพิ่ม security headers:
  ```nginx
  add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; font-src 'self'; connect-src 'self' https://api.line.me" always;
  add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;
  ```
- เพิ่ม `error_page` directives + custom error page
- **`nginx/50x.html`**: สร้าง custom error page HTML ง่ายๆ
- **`nginx/Dockerfile`**: COPY 50x.html เข้าไป

#### C6: Update root `.env.example`
- คัดลอกจาก `.env.production.example` + แก้ dev values (localhost, false, etc.)
