# Phase 1 — Admin Dashboard Foundation

## Scope
1. สร้าง `apps/admin/` — โครงสร้าง monorepo app ใหม่
2. ย้าย API response types เข้า `@auan/types`
3. Admin Login page — Staff Auth (email+password)
4. Admin Layout — Sidebar + Top Navigation + Breadcrumb
5. Dashboard page — Stats, Revenue, Popular Products, Latest Orders, Quick Actions
6. Shared components ที่จำเป็น — Button, Input, Card, Badge, Dialog, Skeleton, Empty State, Status Badge, Data Table

## Step 3: Admin Auth (Staff Auth)

**Pattern:** เหมือน customer app — React Context + localStorage + TanStack Query mutations

**Auth flow:**
1. `POST /api/v1/auth/staff/login` → `{ email, password }` → `{ accessToken, sessionToken, expiresIn, staff: { id, email, displayName, role } }`
2. เก็บ `admin_access_token`, `admin_session_token`, `admin_staff_id`, `admin_staff_role`, `admin_display_name` ใน localStorage
3. API client ส่ง `Authorization: Bearer <admin_access_token>`
4. Refresh: `POST /api/v1/auth/staff/refresh` → `{ sessionToken }` → `{ accessToken, sessionToken, expiresIn }`
5. Logout: `POST /api/v1/auth/staff/logout` + clear localStorage

**Login page:**
- Form: email (required, valid email), password (required, min 8 chars)
- Zod validation, React Hook Form
- Error display from API
- Loading state on submit button
- รองรับ keyboard (Enter = submit)

**Route guards:**
- `ProtectedRoute`: ไม่มี token → redirect `/login`
- `PublicRoute`: มี token → redirect `/`

## Step 6: Shared UI Components

สร้าง components ที่จำเป็นด้วย CVA pattern (เหมือน customer app):

### `button.tsx`
- Variants: `default` | `secondary` | `outline` | `ghost` | `destructive`
- Sizes: `sm` | `md` | `lg`
- States: loading (spinner + disabled), disabled
- forwardRef pattern

### `input.tsx`
- Label integration
- Error display
- forwardRef + `useId()`

### `card.tsx`
- Variants: `default` | `metric` | `bordered`
- CVA styling

### `badge.tsx`
- Variants ตรงกับ status colors: `default` | `secondary` | `success` | `warning` | `error` | `info` | `outline`

### `dialog.tsx` (Radix)
- AlertDialog pattern
- Confirmation dialog

### `status-badge.tsx` (admin-specific)
- Maps OrderStatus / PaymentStatus → color variant อัตโนมัติ ตาม `189-design-tokens.md`:
  - Pending → warning (yellow)
  - Preparing → info (blue)
  - Ready → success (green)
  - Completed → success (green)
  - Cancelled → default (gray)
  - Rejected → error (red)

### `data-table.tsx`
- Generic `<DataTable<T>` component
- Column definitions, sorting, pagination props
- จะขยายใน Phase 2

## Conventions ที่ต้องทำตาม

- **Imports order**: Node modules → external libs → @auan/* → relative
- **File naming**: kebab-case (`.tsx`), PascalCase (component export)
- **Component max**: 300 lines
- **Hook max**: 200 lines
- **No `any`** — ใช้ `unknown`
- **Business logic** อยู่ใน hooks/services ไม่ใช่ JSX
- **API calls** ไม่เรียกตรงจาก component — ผ่าน TanStack Query hooks
- **Barrel exports**: ทุก directory มี `index.ts`
