# Fix Plan — Render Crash + Code Audit

## Root Cause 1: Render Crash 🔴 CRITICAL

**Error:** `data/required must be array` ทุก route ที่มี `schema`

**สาเหตุ:** `fastify-type-provider-zod` ติดตั้งอยู่แต่ **ไม่ได้ใช้งาน** — server.ts สร้าง `Fastify({})` แบบ plain โดยไม่มี `validatorCompiler`, `serializerCompiler`, หรือ swagger `transform`

Zod schemas ถูกส่งเข้า Fastify โดยตรง → Fastify คาดว่าเป็น JSON Schema → parse ไม่ได้ → crash

---

## Fix Step 1: ตั้งค่า Zod Type Provider (ที่ `server.ts`)

**แก้ไฟล์:** `apps/api/src/server.ts`

```ts
import fastifyTypeProviderZod from "fastify-type-provider-zod"
import { ZodTypeProvider } from "fastify-type-provider-zod"

const app = Fastify({
  logger: { ... },
  requestIdHeader: "x-request-id",
}).withTypeProvider<ZodTypeProvider>()
```

เพิ่ม validator/serializer compilers:

```ts
const zodProvider = fastifyTypeProviderZod()
app.setValidatorCompiler(zodProvider.validatorCompiler)
app.setSerializerCompiler(zodProvider.serializerCompiler)
```

---

## Fix Step 2: เพิ่ม Swagger `transform` (ที่ `plugins/swagger.ts`)

Swagger ต้องมี `transform` เพื่อ convert Zod → JSON Schema สำหรับ OpenAPI docs

```ts
import { jsonSchemaTransform } from "fastify-type-provider-zod"

await app.register(fastifySwagger, {
  openapi: { ... },
  transform: jsonSchemaTransform,  // ← เพิ่ม
})
```

## Fix Step 4: LINE Webhook HMAC Bug 🔴 CRITICAL

**ไฟล์:** `modules/line-webhook/line-webhook.controller.ts`

**ปัญหา:** `JSON.stringify(body)` ทำให้ byte sequence เปลี่ยน → signature verify fail ตลอด

**แก้:** ใช้ Fastify `rawBody` option เพื่ออ่าน body ก่อน parse:

```ts
// ใน route definition:
app.post("/webhooks/line", {
  schema: ...,
  bodyLimit: 1024 * 1024,
  // ต้องอ่าน raw request body ก่อน Fastify parse
  handler: ...,
})
```

หรือใช้ `request.rawBody` หาก fastify เปิด `rawBody` option
